# 区块链溯源系统 - 项目文档

## 概述

本系统是一个完整的区块链溯源解决方案，使用Go语言开发，旨在提供产品全生命周期的可追溯性、透明性和数据完整性。系统支持P2P网络通信、智能缓存、图像处理与目标检测、用户认证授权等功能，并提供Web界面和命令行客户端两种使用方式。

## 系统架构

系统采用客户端-服务器架构，包含以下主要组件：

- **客户端组件**：包含Web界面和命令行工具，负责用户交互和数据展示
- **P2P网络**：支持节点间直接通信，实现区块链数据同步
- **主节点系统**：负责区块链数据的存储和管理
- **缓存系统**：提升性能和减少网络请求
- **图像处理系统**：集成YOLOv8模型进行产品质量检测

## 核心功能

### 1. 区块链溯源功能

- **批次管理**：创建和管理产品批次，为每个批次分配唯一标识
- **流程记录**：记录产品在各环节的处理信息，包括处理者、时间、地点、详情等
- **多级溯源**：支持多级供应链溯源（来源、一级加工厂、二级加工厂等）
- **数据不可篡改**：利用区块链技术确保数据一旦记录不可篡改
- **备注与图片**：支持为每个处理环节添加文字备注和图片证据

### 2. P2P网络通信

- **节点发现与管理**：自动发现和管理网络中的节点
- **区块链同步**：支持轻节点同步、剪枝节点同步和全节点同步三种模式
- **主节点选择**：基于网络延迟或权重的主节点选择策略
- **自动重连**：连接断开后自动尝试重新连接
- **握手认证**：节点间通信前进行握手认证

### 3. 智能缓存系统

- **多级缓存**：分别缓存批次列表、查询结果和QR码
- **TTL管理**：支持不同数据类型的自定义过期时间
- **LRU策略**：自动清理最久未使用的缓存条目
- **实时统计**：提供缓存命中率、大小等关键指标
- **手动管理**：支持手动清除缓存和刷新数据

### 4. YOLOv8图像处理

- **图像上传**：支持上传产品相关图片
- **目标检测**：集成YOLOv8模型进行产品质量检测
- **结果管理**：保存和展示检测结果，包括检测到的对象、置信度和边界框
- **结果上传**：支持将检测结果上传至区块链系统

### 5. 认证与授权

- **用户注册与登录**：支持用户账号的创建和身份验证
- **JWT令牌**：使用JSON Web Token进行会话管理
- **角色权限**：不同用户角色拥有不同的操作权限
- **API保护**：关键API端点需要认证才能访问

## 技术规格

### 网络配置

- **HTTP服务端口**：5500
- **P2P服务端口**：3000
- **同步间隔**：24小时（仅在启动时同步一次）
- **重连延迟**：5秒
- **主节点列表**：支持多个主节点配置，自动选择最优节点

### 缓存配置

- **批次列表缓存**：5分钟
- **查询结果缓存**：10分钟  
- **QR码缓存**：30分钟
- **清理间隔**：2分钟
- **最大缓存条目**：100条

### 支持的主节点

- 主节点1：111.230.7.188（HTTP端口8080，P2P端口3000）
- 主节点2：152.136.16.231（HTTP端口8080，P2P端口3000）

## 使用方法

### 启动系统

```bash
# 直接运行
go run main.go

# 编译后运行
$env:GOOS="windows"
go build -ldflags="-H=windowsgui -s -w" -o myapp.exe main.go
./myapp.exe
```

系统启动后会自动：
1. 初始化P2P网络连接
2. 连接到主节点并同步区块链数据
3. 启动本地HTTP服务器（http://localhost:5500）
4. 自动打开浏览器访问Web界面

### Web界面操作

1. **登录系统**：使用注册的账号登录系统
2. **查看批次列表**：在首页查看所有可追溯的批次
3. **查看产品详情**：点击批次查看详细的处理流程和信息
4. **执行同步操作**：可以选择执行轻节点、剪枝节点或全节点同步
5. **上传图片**：为处理环节上传相关图片
6. **进行检测**：使用YOLOv8模型进行产品质量检测

### API接口

系统提供以下主要API接口：

- **认证接口**：
  - `/api/auth/login`：用户登录
  - `/api/auth/register`：用户注册

- **区块链操作接口**：
  - `/api/batch/register`：注册新批次
  - `/api/process`：添加处理记录
  - `/api/product/{batch_id}`：查询产品信息
  - `/api/batches`：获取批次列表

- **同步接口**：
  - `/api/sync/light`：执行轻节点同步
  - `/api/sync/pruned`：执行剪枝节点同步
  - `/api/sync/full`：执行全节点同步

- **图像处理接口**：
  - `/api/upload/image`：上传图片
  - `/detect`：执行图像检测

## 缓存管理

### Web界面缓存管理

在Web界面中，您可以通过专门的"缓存管理"卡片执行以下操作：

- **查看缓存统计**：实时显示缓存命中次数、未命中次数、当前缓存大小
- **检查批次列表缓存状态**：查看批次列表缓存是否有效
- **清除所有缓存**：一键清除所有缓存数据
- **刷新批次列表**：强制刷新批次列表数据
- **查看缓存详细信息**：查看缓存的具体内容和状态

### 缓存自动管理

- 缓存统计每5秒自动更新
- 后台自动清理过期的缓存条目
- 成功操作后自动清除相关缓存，确保数据新鲜度
- 缓存达到容量限制时自动清理最旧的条目

## 故障排除

### 常见问题

1. **无法连接主节点**
   - 检查网络连接是否正常
   - 确认防火墙是否阻止了P2P端口(3000)的通信
   - 查看日志中的具体错误信息

2. **同步失败**
   - 检查P2P网络连接状态
   - 确认使用的是P2P端口(3000)而非HTTP端口(8080)
   - 尝试手动执行同步操作

3. **缓存不生效**
   - 检查缓存配置是否正确
   - 尝试手动清除所有缓存后重新加载
   - 查看系统日志中的缓存相关信息

4. **检测功能异常**
   - 检查图片格式和大小是否符合要求
   - 确认YOLO服务是否可用
   - 查看检测日志获取详细错误信息

## 性能优化建议

1. **合理使用缓存**：根据数据更新频率设置合适的缓存TTL
2. **监控系统状态**：定期查看缓存命中率和网络连接状态
3. **选择合适的同步模式**：
   - 轻节点同步：快速获取概览信息
   - 剪枝节点同步：获取主要数据但节省带宽
   - 全节点同步：获取完整数据但消耗更多资源
4. **定期维护**：定期清理过期数据，保持系统高效运行
